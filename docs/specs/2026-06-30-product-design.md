# 家庭金融助手 — 产品设计文档

> 状态：已确认 | 日期：2026-06-30

## 一、产品定位

个人金融操作中枢，覆盖**打新信息收集 → 交易操作提醒 → 流水记账分析**三个环节，PC 端和手机端均可使用。

---

## 二、目标用户

- 本人使用，不涉及多用户
- 场景：北交所 + 港股打新相关操作，以及日常转账、积分检查等零散金融任务
- 当前痛点：所有待办靠脑子记，容易遗漏

---

## 三、技术选型

| 层级 | 选择 | 说明 |
|------|------|------|
| 前端框架 | Next.js 14 (App Router) | 响应式，一套代码双端通用 |
| 移动端 | PWA | 添加到桌面后可收推送通知 |
| 数据库 | PostgreSQL (Supabase 免费层) | 云托管，500MB 免费，够用 |
| 部署 | Vercel (免费层) | 含 Cron Jobs，零运维成本 |
| 图表 | ECharts | 流水可视化 |
| AI | LLM API（自然语言解析） | 提醒创建时解析自由文本 |
| 登录 | 访问码 | 设环境变量，输入正确后 Cookie 保持登录 |

## 四、模块设计

### 4.1 IPO 信息面板

- **数据获取**：Vercel Cron Job 每交易日 9:00 自动抓取北交所 + 港股公开新股申购信息
- **展示字段**：公司名称、申购代码、发行价区间、一手资金门槛、保荐人、行业分类、申购截止时间、预计上市日期
- **交互**：
  - 一键创建"申购提醒"
  - 标记关注（收藏列表）
  - 历史新股复盘（已上市/已截止/进行中）
- **容错**：港股数据源准备 2-3 个备用源，主源失败自动切换

### 4.2 提醒管理

#### 创建方式

**A. 模板快速创建** — 6 类预设模板：

| 模板 | 预填字段 |
|------|---------|
| 🏦 申购 | 券商、新股名称、申购代码、截止时间 |
| 💰 卖出 | 券商、新股名称、卖出时间 |
| 💳 转账 | 付款卡尾号、收款人、金额、日期 |
| 🎁 积分 | 平台、检查项 |
| 📊 检查 | 自由检查内容 |
| 📝 自定义 | 完全自由文本 |

**B. 自然语言输入** — 单行文本框，LLM 解析时间/动作/对象/金额，确认后入库。

#### 重复规则

| 模式 | 说明 |
|------|------|
| 仅一次 | 执行后自动归档 |
| 每日重复 | 每个交易日/每天 |
| 每周重复 | 指定周几 |

#### 列表视图

- 今天待执行（按时间排序）
- 未来待执行
- 已完成（归档折叠）

#### 推送机制

- **调度**：Vercel Cron 每分钟轮询待推送提醒，匹配时间 ±1 分钟触发
- **浏览器通知**：PWA Notification API，手机添加到桌面后进入系统通知栏，PC 弹 Chrome 通知
- **微信推送**：通过 PushPlus / Server酱 Webhook 推送到微信
- **容错**：过期未推送的提醒，下次打开页面时补推汇总

### 4.3 流水台账

#### 账户设置

- 用户手动维护账户列表（银行卡尾号、券商名称等），每账户标注类型和备注
- 账户之间转账记录可关联

#### 记账

- **极简录入**：日期（默认今天）、从/到账户、金额、类别、备注
- **智能联动**："从"选好后自动推测"到"（基于历史记录）
- **批量模式**："记录并继续"保持上下文，金额自动清空
- **类别体系**：打新、转账入金、新股收益、消费、利息/分红、其他

#### 列表与筛选

- 按月默认视图，支持按账户、类别、日期范围筛选
- 月度汇总：流入/流出/净额
- 支持**编辑和删除**已录入条目

#### 图表看板

| 图表 | 内容 |
|------|------|
| 月度流水瀑布 | 每月资金流向 + 各账户余额变化 |
| 分类占比饼图 | 各类别支出/收入占比 |
| 账户活跃度 | 各账户转账频次和金额汇总 |

图表支持月/季度/年切换。

### 4.4 设置

- **账户管理**：维护卡和券商账户
- **推送设置**：微信 Token 绑定、浏览器通知授权状态、测试推送
- **IPO 设置**：提醒时间、关注列表
- **数据管理**：JSON/CSV 导出、导入、清空（红色按钮需确认）

| id | uuid | 主键 |

| type | text | 类型（储蓄卡/券商/信用卡） |
| note | text | 备注 |
| created_at | timestamptz | |

**reminders** — 提醒

| description | text | 详细描述 |
| trigger_time | timestamptz | 触发时间 |

| created_at | timestamptz | |

| note | text | 备注 |
| created_at | timestamptz | |

**ipo_listings** — IPO 新股（系统自动抓取）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |

| price_low | decimal | 发行价下限 |

| expected_listing_date | date | 预计上市 |
| status | text | 进行中/已截止/已上市 |
| created_at | timestamptz | |

**watched_ipos** — IPO 关注列表
| 字段 | 类型 | 说明 |
|------|------|------|

| ipo_id | uuid → ipo_listings | |

## 六、部署与运维

- 部署平台：Vercel
- 数据库：Supabase PostgreSQL
- 代码仓库：GitHub
- 环境变量：
  - `ACCESS_CODE` — 登录访问码
  - `SUPABASE_URL` / `SUPABASE_KEY` — 数据库连接
  - `PUSHPLUS_TOKEN` — 微信推送
  - `OPENAI_API_KEY` — LLM 自然语言解析
  - `CRON_SECRET` — Cron Job 鉴权

## 七、不做的

- ❌ 不接入真实银行 API / 券商 API

- ❌ 不做自动交易 / 自动转账
- ❌ 深度 IPO 分析（超额认购倍数、孖展、暗盘等）
